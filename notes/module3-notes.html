Foundations of Number Theory

Cryptography is built upon the hard mathematical problems of number theory. This module provides the essential toolbox: from prime numbers to modular arithmetic and discrete logarithms. Understanding these concepts is crucial for grasping public‑key cryptosystems like RSA, Diffie‑Hellman, and elliptic curve cryptography.

Prime and Relatively Prime Numbers

A prime number is an integer greater than 1 that has no positive divisors other than 1 and itself. Examples: 2, 3, 5, 7, 11, … A composite number has more than two factors. The Fundamental Theorem of Arithmetic states that every integer greater than 1 can be expressed uniquely as a product of primes (up to ordering).

Two integers a and b are said to be relatively prime (or coprime) if their greatest common divisor is 1, i.e. gcd(a, b) = 1. For example, 8 and 9 are coprime, while 8 and 12 are not (gcd = 4).

Modular Arithmetic

Given an integer n > 1, we say two integers a and b are congruent modulo n, written a ≡ b (mod n), if n divides (a – b). This is equivalent to a and b having the same remainder when divided by n.

The set of integers modulo n, denoted ℤn = {0, 1, 2, …, n‑1}, forms a ring under addition and multiplication modulo n. The multiplicative inverse of a modulo n exists if and only if gcd(a, n) = 1. It can be found using the Extended Euclidean Algorithm.

Fermat’s Little Theorem

If p is a prime and a is an integer not divisible by p, then

ap–1 ≡ 1 (mod p)

For example, let p = 7 and a = 3. Then 36 = 729 ≡ 1 (mod 7). Fermat’s theorem is the basis of many primality tests and underlies the correctness of RSA decryption.

Euler’s Theorem

Euler’s totient function φ(n) counts the positive integers up to n that are relatively prime to n. For a prime p, φ(p) = p – 1. For a product of two distinct primes p and q, φ(pq) = (p–1)(q–1).

Euler’s theorem generalises Fermat’s little theorem: if gcd(a, n) = 1, then

aφ(n) ≡ 1 (mod n)

This theorem is essential in RSA because it guarantees that encryption and decryption are inverse operations when working in the multiplicative group modulo n.

Testing for Primality

A primality test determines whether a given integer is prime. Two main approaches exist:

  • Deterministic tests (e.g., trial division, AKS) – guarantee a correct answer but are often slow for large numbers.
  • Probabilistic tests (e.g., Miller–Rabin) – provide a high probability of correctness; widely used in practice.

The Miller–Rabin test is based on Fermat’s little theorem and modular exponentiation. For a candidate n, it checks whether a randomly chosen base a satisfies certain congruences. After enough rounds with different bases, the probability of a composite passing becomes negligible.

Euclid’s Algorithm

The Euclidean algorithm efficiently computes the greatest common divisor (gcd) of two integers. It is based on the observation that gcd(a, b) = gcd(b, a mod b). 

// Iterative implementation
int gcd(int a, int b) {
    while (b != 0) {
        int temp = b;
        b = a % b;
        a = temp;
    }
    return a;
}

The Extended Euclidean algorithm also finds integers x and y such that a·x + b·y = gcd(a, b). This is crucial for computing modular inverses.

The Chinese Remainder Theorem (CRT)

The CRT solves systems of congruences with pairwise coprime moduli. If we have: 

x ≡ a1 (mod m1)
x ≡ a2 (mod m2)
…
x ≡ ak (mod mk)

and the moduli mi are pairwise coprime, then there exists a unique solution modulo M = m1·m2·…·mk.

The solution is constructed using the formula:
x = (a1·M1·y1 + a2·M2·y2 + …) mod M, where Mi = M / mi and yi is the modular inverse of Mi modulo mi.

CRT is used to speed up RSA decryption (by computing modulo p and q separately) and in several cryptographic protocols.

Discrete Logarithms

Given a prime p, a generator (primitive root) g of the multiplicative group ℤp*, and an element h in the group, the discrete logarithm problem is to find an integer x such that gx ≡ h (mod p).

While modular exponentiation is easy (fast exponentiation), computing the discrete logarithm is believed to be computationally hard. This asymmetry is the foundation of the Diffie–Hellman key exchange, the Digital Signature Algorithm (DSA), and ElGamal encryption.

The same concept applies to elliptic curves, where the discrete logarithm problem in the group of points on a curve forms the basis of ECC.

💡 Summary: Modular arithmetic, prime numbers, and the hardness of problems like factoring and discrete logarithms are the building blocks of modern cryptographic systems. A solid grasp of these number‑theoretic concepts is essential for understanding the security of the algorithms discussed in later modules.
