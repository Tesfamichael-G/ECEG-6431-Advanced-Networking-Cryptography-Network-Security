The Unique Challenges of Wireless Security

Wireless networks broadcast data over radio waves, making them inherently more vulnerable than wired networks. Attackers can eavesdrop without physical access, inject frames, or mount denial‑of‑service attacks from a distance. This module explores the security mechanisms used in Wireless LANs (Wi‑Fi), cellular networks, and emerging ad‑hoc and sensor networks.

Security in Wireless LANs (IEEE 802.11)

Wi‑Fi security has evolved through several generations, each addressing the weaknesses of its predecessor.

WEP (Wired Equivalent Privacy) – Broken Legacy

WEP was the first security standard for 802.11, using the RC4 stream cipher with a 40‑bit or 104‑bit key concatenated with a 24‑bit Initialisation Vector (IV). Severe design flaws led to its complete compromise:

  • Short IVs (24 bits) repeat quickly, allowing key recovery through statistical attacks (Fluhrer–Mantin–Shamir).
  • Weak integrity check (CRC‑32) enables packet forgery.
  • No key management – keys are static and shared among all users.
⚠️ Never use WEP. It can be cracked in minutes with tools like Aircrack‑ng.

WPA (Wi‑Fi Protected Access) – The Interim Fix

WPA was introduced as an emergency upgrade while the 802.11i standard was being finalised. It retained RC4 but introduced TKIP (Temporal Key Integrity Protocol):

  • Per‑packet key mixing (using a 48‑bit IV).
  • Michael message integrity code (MIC) replacing CRC‑32.
  • Dynamic key generation via 802.1X/EAP (WPA‑Enterprise) or a Pre‑Shared Key (PSK) (WPA‑Personal).

TKIP was designed as a software upgrade for older hardware, but it is now deprecated due to cryptographic weaknesses.

WPA2 (IEEE 802.11i) – The Robust Standard

WPA2 mandates CCMP (Counter Mode with CBC‑MAC Protocol), based on the AES block cipher. CCMP provides:

  • Data confidentiality (AES‑CTR).
  • Integrity and origin authentication (CBC‑MAC).
  • Replay protection (packet number).

WPA2‑Personal uses a 4‑way handshake to derive per‑session keys from a PSK. WPA2‑Enterprise integrates with a RADIUS server for 802.1X authentication, allowing unique credentials per user.

WPA3 – The Current Standard

WPA3 brings major security improvements over WPA2:

  • Simultaneous Authentication of Equals (SAE) – a password‑authenticated key exchange resistant to offline dictionary attacks (Dragonfly protocol).
  • Forward secrecy – even if the password is later compromised, past session keys cannot be recovered.
  • Protected Management Frames (PMF) – mandatory, preventing deauthentication attacks.
  • Wi‑Fi Enhanced Open – opportunistic encryption for open networks (OWE).

802.1X and EAP

IEEE 802.1X provides port‑based network access control. Before a client can send data, it must authenticate. The architecture involves:

  • Supplicant – the client device.
  • Authenticator – the access point or switch.
  • Authentication Server – typically a RADIUS server.

The Extensible Authentication Protocol (EAP) carries the authentication messages. Common EAP methods: PEAP, EAP‑TLS, EAP‑TTLS, EAP‑SIM.

Security in Cellular Networks

Cellular networks have evolved from analogue (1G) to all‑IP (5G), with security improving at each generation.

2G (GSM) Security

  • A3/A8/A5 algorithms – used for authentication, key generation, and encryption (A5/1 stream cipher).
  • Weaknesses: no mutual authentication (only network authenticates phone); fake base stations (IMSI catchers) possible; A5/1 later broken.

3G (UMTS) Security

  • Mutual authentication – the phone also authenticates the network.
  • Integrity protection for signalling messages.
  • Stronger encryption (KASUMI block cipher) with longer keys.

4G (LTE) and 5G Security

  • Stronger cryptographic algorithms (AES‑based, SNOW 3G, ZUC).
  • Hierarchical key derivation with forward/backward secrecy between handovers.
  • 5G introduces SUCI (Subscription Concealed Identifier) to protect the permanent identity (SUPI/IMSI) from being sent in clear text.
  • SEAF (Security Anchor Function) and Authentication Server Function (AUSF) as part of the 5G core.

Ad‑hoc and Sensor Network Security

Mobile Ad‑hoc Networks (MANETs)

MANETs lack fixed infrastructure; nodes act as routers. This dynamic, self‑organising nature creates unique security challenges:

  • No central authority – distributed trust models needed.
  • Routing attacks: black hole, wormhole, sinkhole, rushing attacks.
  • Sybil attack – a single node forges multiple identities to subvert routing or data aggregation.
  • Resource constraints – battery and bandwidth limit the use of heavyweight cryptography.

Wireless Sensor Networks (WSNs)

WSNs consist of many small, low‑power sensor nodes that collect environmental data. Their security is particularly challenging due to extreme resource limitations:

  • Key management: pre‑distribution schemes (e.g., random key pre‑distribution) allow nodes to establish pair‑wise keys after deployment.
  • Lightweight cryptography: algorithms like PRESENT, SPECK, and ASCON are designed for constrained devices.
  • Attacks: node capture, sleep deprivation, side‑channel analysis, physical tampering.
  • Authentication and integrity – often rely on symmetric MACs and hash functions due to the high cost of public‑key operations.

Defence Mechanisms

  • Intrusion Detection Systems (IDS) – monitor for anomalous behaviour (e.g., sudden routing changes).
  • Secure routing protocols (e.g., Ariadne, SEAD for MANETs) that incorporate authentication.
  • Tamper‑proof packaging and secure hardware modules for sensors.
  • Redundancy and diversity to cope with compromised nodes.

Summary

Wireless security has come a long way from the broken WEP to the robust, forward‑secret WPA3. Cellular networks continue to strengthen their security architectures, while ad‑hoc and sensor networks remain an active research frontier.
