What is a Socket?

A socket is a software endpoint that enables communication between two processes over a network. It is identified by a combination of an IP address and a port number. Sockets abstract the underlying transport‑layer protocols (TCP or UDP) and provide a standard interface for sending and receiving data.

Types of Sockets

  • Stream Sockets (TCP) – connection‑oriented, reliable, ordered, and error‑checked. Data is delivered as a continuous stream of bytes without message boundaries.
  • Datagram Sockets (UDP) – connectionless, best‑effort delivery, unordered, and preserves message boundaries. No guarantee of delivery or order.
  • Raw Sockets – allow direct access to lower‑layer protocols (e.g., ICMP, IGMP). Typically used for network diagnostics and custom protocol implementations.

The Client‑Server Model

In a client‑server architecture, the server listens on a well‑known port, waiting for incoming connection requests. The client initiates the communication by connecting to the server’s IP address and port. Once the connection is established, data flows bidirectionally.

Socket Interface and TCP Connection Features

The standard POSIX socket API (used in C) and its Java counterpart provide a set of functions to create and manage sockets.

  • socket() – creates a socket descriptor.
  • bind() – assigns a local address and port to a socket.
  • listen() – marks a socket as passive to accept connections (server only).
  • accept() – accepts an incoming client connection and returns a new socket for communication.
  • connect() – initiates a connection to a remote server (client).
  • send()/recv() – transmit and receive data (TCP).
  • sendto()/recvfrom() – used for UDP datagrams.

A TCP connection is full‑duplex, reliable, and provides flow control and congestion control.

Byte Ordering

Different CPU architectures store multi‑byte integers differently: little‑endian (least significant byte first) vs. big‑endian (most significant byte first). To ensure interoperability, network protocols use a standard big‑endian (network byte order).

Functions like htons(), htonl(), ntohs(), and ntohl() convert between host byte order and network byte order for 16‑bit and 32‑bit values respectively.

💡 Always convert multi‑byte data to network byte order before sending, and back to host byte order upon receipt.

Address Structures

The standard address structure for IPv4 in C is sockaddr_in:

struct sockaddr_in {
    sa_family_t    sin_family;  // AF_INET
    in_port_t      sin_port;    // 16‑bit port number (network byte order)
    struct in_addr sin_addr;    // 32‑bit IPv4 address (network byte order)
    char           sin_zero[8]; // padding
};

For IPv6, sockaddr_in6 is used. In Java, the InetAddress class abstracts this complexity.

Ports and Address Conversion Functions

Port numbers (0–65535) identify specific services; well‑known ports are 0‑1023. inet_pton() converts a textual IP address to binary network format, while inet_ntop() performs the reverse. Older functions (inet_addr(), inet_ntoa()) are deprecated.

Outline of a TCP Server

  1. Create a socket with socket().
  2. Bind the socket to a local address and port using bind().
  3. Mark the socket as passive with listen().
  4. Accept a client connection with accept(), which returns a new socket.
  5. Exchange data using send() and recv().
  6. Close the connection with close().

Outline of a TCP Client

  1. Create a socket with socket().
  2. Connect to the server’s address and port using connect().
  3. Exchange data using send() and recv().
  4. Close the connection with close().

Client‑Server Communication Flow

A typical exchange: the client sends a request (e.g., an HTTP GET), the server processes it and returns a response (e.g., a web page). This request‑response cycle repeats until one side closes the connection. In a multi‑client scenario, the server typically forks a new process or thread for each client.

Practical Example (C Skeleton)

// Server snippet
int server_fd = socket(AF_INET, SOCK_STREAM, 0);
struct sockaddr_in addr;
addr.sin_family = AF_INET;
addr.sin_port = htons(8080);
inet_pton(AF_INET, "0.0.0.0", &addr.sin_addr);
bind(server_fd, (struct sockaddr*)&addr, sizeof(addr));
listen(server_fd, 5);
int client_fd = accept(server_fd, NULL, NULL);
// ... receive / send data ...
close(client_fd);
close(server_fd);
