System Security: The Big Picture

Even with strong cryptographic protocols, a system can be compromised through software vulnerabilities, human factors, or weak perimeter defences. This module addresses three critical areas of system‑level security: understanding intruders, defending against malicious software (malware), and deploying firewalls to control network access.

Intruders

An intruder is an individual or automated entity that attempts to gain unauthorised access to a system or network. Intruders are typically classified into three categories:

  • Masquerader – an outsider who pretends to be an authorised user (e.g., using stolen credentials).
  • Misfeasor – a legitimate user who accesses unauthorised data or misuses privileges.
  • Clandestine user – an individual who seizes supervisory control to evade audit and access controls.

Intrusion Detection Systems (IDS) are designed to detect such activities. Two main approaches exist:

  • Signature‑based detection – matches known attack patterns (signatures) against observed events. Effective for known threats, but cannot detect new attacks.
  • Anomaly‑based detection – builds a profile of normal behaviour and flags deviations. Can detect novel attacks but may generate false positives.

Intrusion Prevention Systems (IPS) extend IDS by automatically blocking detected threats. Honeypots are decoy systems that lure attackers, allowing security teams to study their techniques without exposing real assets.

Malicious Software (Malware)

Malware is any software intentionally designed to cause damage, disrupt operations, or gain unauthorised access. Common categories include:

Viruses

A virus attaches itself to a legitimate program or file and replicates when that program is executed. Types include boot sector viruses, file infectors, macro viruses, and polymorphic viruses that change their code signature to evade detection.

Worms

Unlike viruses, worms spread without user interaction by exploiting network vulnerabilities. Famous examples: Morris Worm (1988), Code Red, SQL Slammer.

Trojan Horses

A trojan disguises itself as useful software (e.g., a game, utility) but delivers a malicious payload. Trojans often create backdoors, download additional malware, or steal data.

Ransomware

Ransomware encrypts the victim’s files and demands a ransom (often in cryptocurrency) for the decryption key. Recent high‑profile variants include WannaCry, Ryuk, and LockBit. Protection involves regular backups and robust endpoint security.

Spyware and Adware

  • Spyware – covertly gathers user information (keystrokes, browsing habits) and transmits it to a third party.
  • Adware – aggressively displays advertisements; may be bundled with free software.

Rootkits

A rootkit provides an attacker with persistent, undetectable presence by modifying the operating system or firmware. Rootkits are extremely difficult to remove; often the safest remedy is a complete system rebuild.

Botnets

A network of compromised devices (bots) controlled by a command‑and‑control (C&C) server. Botnets are used for distributed denial‑of‑service (DDoS) attacks, spam campaigns, and credential theft. They can consist of millions of devices, including IoT gadgets.

⚠️ Defence in Depth: No single solution stops all malware. Effective defence combines antivirus, endpoint detection and response (EDR), firewalls, intrusion prevention, user education, and regular patching.

Firewalls

A firewall is a network security device (hardware, software, or virtual) that monitors and controls incoming and outgoing traffic based on a predefined security policy. It forms the first line of defence between a trusted internal network and an untrusted external network.

Types of Firewalls

  • Packet Filtering Firewall – inspects packets based on source/destination IP, port, and protocol. Fast but lacks context; vulnerable to IP spoofing.
  • Stateful Inspection Firewall – tracks the state of active connections and makes decisions based on context. More secure than simple packet filters.
  • Application‑Level Gateway (Proxy Firewall) – acts as an intermediary for specific applications (HTTP, FTP). Can inspect payload content but introduces latency.
  • Circuit‑Level Gateway – validates TCP/UDP sessions without inspecting the data itself; often used alongside application proxies.
  • Next‑Generation Firewall (NGFW) – combines stateful inspection with intrusion prevention, application awareness, and deep packet inspection.

Firewall Architectures

  • Dual‑homed host – a single host with two network interfaces, one connected to the internal network and one to the external network. Traffic must pass through the host, where firewall rules are enforced.
  • Screened host – a packet‑filtering router sits in front of a bastion host (hardened proxy server).
  • Screened subnet (DMZ) – a Demilitarized Zone isolates publicly accessible servers (web, mail) from the internal network. Two firewalls or a multi‑homed firewall separate the internal network, the DMZ, and the Internet.

Firewall Policy

A firewall's behaviour is governed by a set of rules. The default stance may be:

  • Default‑deny – block all traffic unless explicitly permitted. Most secure, but may disrupt legitimate services if not carefully configured.
  • Default‑allow – permit all traffic unless specifically denied. Less secure, sometimes used in legacy environments.

Beyond Firewalls: Defense in Depth

Modern system security layers multiple defences:

  • Network segmentation – dividing a network into smaller segments limits an attacker’s lateral movement.
  • Zero Trust Architecture – no entity (inside or outside) is trusted by default; all access requires continuous verification.
  • Endpoint Protection – antivirus, EDR, application whitelisting.
  • Security Information and Event Management (SIEM) – aggregates logs and alerts for centralised analysis.
  • Regular patching – closing known vulnerabilities remains one of the most effective defences.

Physical Security

No amount of cryptography or firewalls will protect a system if an attacker can physically access the hardware. Physical security measures include:

  • Access controls (locks, biometrics, security guards).
  • Video surveillance and intrusion alarms.
  • Hardware‑based encryption (self‑encrypting drives).
  • Secure disposal of storage media (degaussing, shredding).

Social Engineering

The weakest link in any security system is often the human. Social engineering attacks manipulate people into divulging confidential information or performing actions that compromise security. Common techniques:

  • Phishing – fraudulent emails or websites that impersonate trusted entities.
  • Pretexting – creating a fabricated scenario to obtain information (e.g., fake tech support).
  • Baiting – offering something enticing (free music, USB drives) that contains malware.
  • Tailgating – following an authorised person into a restricted area without proper credentials.

Mitigation requires continuous security awareness training and a culture that encourages reporting suspicious activity.

Summary

Securing a system requires a holistic approach: detecting and deterring intruders, defending against a diverse range of malware, deploying robust firewall architectures, and never forgetting the human element. With the knowledge from this module and the entire course, you now have the foundation to design, evaluate, and maintain secure networked systems.
